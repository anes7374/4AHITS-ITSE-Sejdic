## 1. Übung (nmap enumeration)

```bash
nmap -Pn -sS -sV -sC -oN nmap_default.txt 192.168.8.51
```

### 1.1 Bedeutung der Optionen

- **-Pn**  
  Überspringt den Ping-Scan und behandelt das Zielsystem als erreichbar, auch wenn es nicht auf ICMP-Anfragen antwortet.

- **-sS**  
  Führt einen SYN-Scan (Half-Open Scan) durch, bei dem keine vollständige TCP-Verbindung aufgebaut wird.

- **-sV**  
  Erkennt die laufenden Dienste inklusive ihrer Versionsnummern.

- **-sC**  
  Führt die Standard-Nmap-Skripte aus und sammelt zusätzliche Informationen über die Dienste.

- **-oN nmap_default.txt**  
  Speichert die Ergebnisse des Scans in einer Textdatei zur Dokumentation.

### 1.2 Besonderer Port-Scan

Es wird ein **SYN-Scan (-sS)** durchgeführt.  
Dabei wird keine vollständige TCP-Verbindung aufgebaut.

**Vorteil dieses Scans**

- Schnell  
- Weniger auffällig  
- Zuverlässige Erkennung offener Ports 

### 1.3 Anzahl der gescannten Ports

Standardmäßig scannt nmap die **1000 häufigsten TCP-Ports**.


### 1.4 Metasploitable Scan
Befehl:
```bash
nmap -Pn -sS -sV -sC -oN nmap_default.txt 192.168.8.51
```

**Ergebnis des Enumeration-Scans**

Der Scan zeigt, dass auf der Metasploitable-VM viele Dienste aktiv sind, darunter mehrere veraltete und unsichere Services.  
Gefunden wurden u. a.:

- FTP (vsftpd 2.3.4, anonymer Login erlaubt)  
- SSH (OpenSSH 4.7p1)  
- Telnet  
- HTTP (Apache 2.2.8)  
- MySQL und PostgreSQL  
- Samba und IRC  

Durch `-sV` wurden die exakten Versionsnummern der Dienste ermittelt.  
Die Standard-Skripte (`-sC`) lieferten zusätzliche Informationen wie Banner, HTTP-Titel und SMB-Daten.

## 2. Vulnerability Scan mit nmap Skripten

### 2.1 Fast-Scan

```bash
nmap -Pn -sS -sV -sC -F -oN nmap_vuln.txt 192.168.8.51
```
Die Option `-F` scannt nur die **100 häufigsten TCP-Ports** und ist dadurch deutlich **schneller**. Trotzdem wurden mehrere **unsichere und veraltete Dienste** gefunden, z. B. FTP mit anonymem Login, Telnet sowie alte SSH- und Apache-Versionen. Der Fast-Scan zeigt bereits eine große **Angriffsfläche** und deutet auf **schwere Sicherheitslücken** hin.

### 2.2 Ausgewählte Skripte

```bash
nmap --script http-title,mysql-info,banner,ftp-anon,ssh-hostkey 192.168.8.51
```

**Verwendete Skripte:**

| Skript        | Zweck |
|---------------|-------|
| http-title    | Titel der Webseite |
| mysql-info    | MySQL Versionsinfos |
| banner        | Dienst-Banner |
| ftp-anon      | Anonymer FTP-Zugang |
| ssh-hostkey   | SSH-Schlüssel |

### 2.3 Weiteres Skript

```bash
nmap --script smb-os-discovery 192.168.8.51
```

Dieses Skript nutzt den SMB-Dienst, um zusätzliche Informationen über das Zielsystem zu sammeln.

Ergebnis:
- Erkennung des verwendeten Betriebssystems
- Anzeige des Rechnernamens und der Workgroup
- Informationen über SMB-Freigaben und Konfiguration

  Z.B. folgende Informationen wurden ermittelt:
- Rechnername: `metasploitable`
- Domain: `localdomain`
- FQDN: `metasploitable.localdomain`

Damit können weitere Details über die Netzwerkumgebung und das Zielsystem ermittelt werden.

### 2.4 Kategorie vuln

```bash
nmap --script=vuln -p 21,80 192.168.8.51
```

Dieser Scan überprüft bekannte Schwachstellen auf FTP und HTTP.

**Ergebnis:**

Auf Port 21 (FTP) wurde eine kritische Schwachstelle gefunden:
- vsftpd 2.3.4 Backdoor (CVE-2011-2523).

Diese Schwachstelle erlaubt das Öffnen einer Root-Shell und ermöglicht somit Remote Command Execution.

Auf Port 80 (HTTP) wurden mehrere mögliche Web-Schwachstellen erkannt, darunter:

- Hinweise auf CSRF-anfällige Formulare
- mögliche SQL-Injection-Stellen
- mögliche Anfälligkeit für Slowloris (DoS, CVE-2007-6750)
- aktiviertes TRACE-Verfahren (unsichere Konfiguration)

Der Scan zeigt, dass sowohl der FTP- als auch der Webdienst schwerwiegende Sicherheitslücken aufweisen.
Besonders kritisch ist die gefundene Backdoor im FTP-Dienst, da sie vollständigen Systemzugriff erlaubt.
