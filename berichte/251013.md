## Lösung / Durchführung

### 1) VM-Setup

* **Kali Linux:** als VM (VirtualBox / VMware) installiert bzw. importiert.
* **Metasploitable:** OVA importiert.
* Beide VMs wurde(n) im Labor-Netz auf **Bridged Adapter** konfiguriert. Falls Bridged mit dem Host-WLAN nicht funktioniert, wurde für Tests temporär **NAT** verwendet.

**Kommandoausgaben (Beispiele):**

```bash
# Prüfen der Netzwerkinterfaces

# Verbindungstest

```

### 2) Netzwerk-Probleme & Behebung

Beim ersten Klonversuch trat folgender Fehler auf:

```bash
$ git clone https://github.com/anes7374/4AHITS-ITSE-Sejdic
Cloning into '4AHITS-ITSE-Sejdic'...
fatal: unable to access 'https://github.com/anes7374/4AHITS-ITSE-Sejdic/': Could not resolve host: github.com
```

**Analyse:** DNS-Auflösung / DHCP-Probleme. Es folgten weitere Diagnoseschritte:

```bash
$ sudo dhclient -v
sudo: dhclient: command not found
```

Lösung(en):

* Entweder `isc-dhcp-client` nachinstallieren: `sudo apt install isc-dhcp-client -y`
* Oder `NetworkManager` nutzen: `nmcli device connect <interface>`
* Falls weiterhin kein DHCP verfügbar, VirtualBox/VMware Netzwerkeinstellungen prüfen (Bridged vs NAT). Bei Bedarf `/etc/resolv.conf` temporär setzen mit `nameserver 8.8.8.8`.

Ein weiterer Fehler bei Verbindungsaktivierung:

```bash
Error: Connection activation failed: IP configuration could not be reserved (no available address, timeout, etc.).
```

**Ursache:** Kein erreichbarer DHCP-Server in diesem Bridged-Setup (häufig bei bestimmten WLAN-Adaptern). Abhilfe: Host-Adapter wechseln oder NAT verwenden.

### 3) Installation Visual Studio Code

Schritte (ausgeführt auf Kali):

```bash
sudo apt update

sudo install -o root -g root -m 644 packages.microsoft.gpg /usr/share/keyrings/
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/packages.microsoft.gpg] https://packages.microsoft.com/repos/code stable main" | sudo tee /etc/apt/sources.list.d/vscode.list
sudo apt update && sudo apt install code -y
# Starten:
code &
```

### 4) Git-Klon und Änderungen

Klon:

```bash
git clone https://github.com/anes7374/4AHITS-ITSE-Sejdic
cd 4AHITS-ITSE-Sejdic
```

Lokale Änderung in `README.md` mit VS Code:

* Ergänzung: `## Änderung vom Kali-System`

Commit & Push:

```bash
git add README.md
git commit -m "Update README.md: Änderung aus Kali"
git push
```

Beim ersten Push trat auf:

```bash
remote: Invalid username or token. Password authentication is not supported for Git operations.
fatal: Authentication failed for 'https://github.com/anes7374/4AHITS-ITSE-Sejdic/'
```

**Lösung:** GitHub erlaubt keine Passwort-Authentifizierung mehr. Zwei sichere Optionen:

* **Personal Access Token (PAT)**: Token in GitHub erzeugen und als Passwort verwenden.
* **SSH-Keys**: SSH-Key erzeugen (`ssh-keygen -t ed25519`), Public Key in GitHub eintragen und Remote-URL auf `git@github.com:...` ändern.

Nach Einrichtung von PAT oder SSH funktionierte `git push`.

### 5) GitHub Pages / HTML Generierung

* Nach dem Push die GitHub-Seite aktualisiert und geprüft, ob die generierte HTML-Seite (GitHub Pages) die Änderung enthält. Hinweis: kurze Wartezeit bis zu wenigen Minuten möglich.

### 6) Änderung via GitHub-Webinterface & Pull

* Über GitHub-Weboberfläche eine Änderung vorgenommen (README bearbeitet und direkt in `main` committed).
* Lokal auf Kali im Repo:

```bash
git pull origin main
# oder
git pull
```

* Prüfung mittels `cat README.md` und `git log --oneline -n 3` bestätigte die Übernahme.
