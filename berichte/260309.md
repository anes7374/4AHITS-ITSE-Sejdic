## 1. VNC Exploitation

### 1.1 VNC Recherche

**VNC (Virtual Network Computing)** ist ein Remote‑Desktop System.\
Damit kann man den Bildschirm eines anderen Computers über ein Netzwerk
sehen und steuern.

#### Funktionsweise

-   basiert auf dem **RFB (Remote Frame Buffer) Protokoll**
-   Client‑Server Modell
-   überträgt:
    -   Bildschirm
    -   Mausbewegungen
    -   Tastatureingaben

#### Komponenten

-   **VNC Server** -- läuft auf dem Zielsystem\
-   **VNC Viewer (Client)** -- verbindet sich zum Server

#### Standard Ports

-   TCP **5900** → Display :0\
-   TCP **5901** → Display :1\
-   TCP **5902** → Display :2

#### Sicherheitsprobleme

-   teilweise **keine Verschlüsselung**
-   **schwache Passwörter**
-   **Brute‑Force Angriffe möglich**
-   alte Versionen können **Authentication Bypass** enthalten


### 1.2 VNC Scan

Zuerst wird überprüft, ob der VNC Port auf dem Metasploitable System offen ist.

```bash
nmap -p 5900 192.168.8.51
```

**Ergebnis**: Port offen

Zusätzlich kann das nmap Script **vnc-info** verwendet werden.

```bash
nmap -p 5900 --script vnc-info <Metasploitable-IP>
```

Dieses Script zeigt zusätzliche Informationen wie:

- VNC Protokollversion
- Security Types
- Authentifizierung

### 1.3 VNC Exploit

Metasploitable enthält eine Schwachstelle im VNC Server.

#### Modul laden

```bash
use auxiliary/scanner/vnc/vnc_login
```

#### Ziel setzen

```bash
set RHOSTS <Metasploitable-IP>
set RPORT 5900
```

#### Scan starten

```bash
run
```
**Ergebnis:**

### 2.2 portmapper/nfs

**NFS (Network File System)** ist ein Protokoll, mit dem ein Computer Dateien über ein Netzwerk freigeben kann.  
Andere Systeme können diese Verzeichnisse einbinden und wie lokale Ordner verwenden.

**Zusammenhang mit Portmapper**

Der **Portmapper (rpcbind)** verwaltet RPC‑Dienste wie NFS und teilt Clients mit, auf welchen Ports diese Dienste laufen.

Wichtige Ports:

| Port | Dienst |
|-----|------|
| 111 | Portmapper / RPC |
| 2049 | NFS |

Wenn ein NFS‑Share falsch konfiguriert ist (z.B. `no_root_squash`), können Angreifer Dateien im System verändern.


Zuerst wird überprüft, ob Portmapper und NFS aktiv sind.

```bash
nmap -p 111,2049 -sV 192.168.8.51
```
--> Beide Ports offen

**NFS Shares anzeigen**

```bash
nmap --script nfs-showmount 192.168.8.51
```

--> Zeigt, welche Ordner sind über NFS freigegeben

**Leider mit der Übung nicht fertig geworden**
